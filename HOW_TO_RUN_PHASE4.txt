EXAMPLE 2: Test Rule Engine
(Can run in Python REPL)

  from backend.app.features.intelligence.rule_engine import RuleEngine
  from backend.app.features.intelligence.models import Rule
  from backend.app.features.transactions.models import Transaction
  from backend.app.features.imports.models import TransactionDirection
  from decimal import Decimal
  from datetime import date
  from uuid import uuid4

  # Create a transaction
  txn = Transaction.create(
      user_id=uuid4(),
      posted_date=date.today(),
      amount=Decimal('100'),
      direction=TransactionDirection.DEBIT,
      merchant='Amazon',
  )

  # Create a rule: Amazon → Shopping category
  rule = Rule.build_exact_match_rule(
      user_id=uuid4(),
      merchant='Amazon',
      category='Shopping',
      priority=10,
  )

  # Evaluate the rule
  engine = RuleEngine()
  match = engine.evaluate_rules(txn, [rule])
  
  print(f"Matched: {match.matched}")  # True
  print(f"Category: {match.action.get('category')}")  # Shopping

  @app.post("/api/transactions/categorize")
  async def categorize_transaction(transaction_id: str, user_id: str):
      """Categorize a transaction using Intelligence Layer."""
      txn = await get_transaction(transaction_id)
      service = CategorizationService()
      
      result = await service.categorize(

          "confidence": result.confidence,
          "confidence_level": result.confidence_level.value,
          "requires_confirmation": result.requires_confirmation(),
          "source": result.source,
      }

SECTION 4: COMPONENT DESCRIPTIONS
═════════════════════════════════════════════════════════════════════════════

What each module does:

models.py
  13 frozen dataclasses defining the domain:
  - Rule, AICategory, Insight, MerchantMemory, etc.
  - Includes builder methods for creating Rules

providers.py
  AI provider abstraction layer:
  - AIProvider base class
  - OpenAI, Gemini, Ollama, Mock implementations
  - All implement same interface (categorize, suggest_merchant, etc.)
  - Mock provider for testing (no API calls)

context_builder.py (CRITICAL FOR PRIVACY)
  Privacy filtering before AI:
  - PrivacyFilter: Detects & redacts sensitive patterns
  - Patterns: Account numbers, phone, UPI, IFSC, card, reference IDs
  - AIContextBuilder: Builds safe context for AI requests
  - Validates all data is sanitized before transmission

rule_engine.py
  Deterministic categorization (Rules → before AI):
  - RuleType enum for 7 rule types
  - RuleEvaluator: Tests if rule matches transaction
  - RuleEngine: Orchestrates rule evaluation with priority
  - Rules can be: exact match, contains, prefix, suffix, regex, amount threshold

merchant_memory.py
  Learning system that improves with time:
  - MerchantMemory: Stores user decisions
  - MerchantMemoryEngine: Apply & find memories
  - Similarity matching: Fuzzy match to handle typos
  - Auto-apply: Applies previous decisions to new transactions

confidence_engine.py
  Confidence scoring for AI suggestions:
  - 3 levels: HIGH (0.85-1.0), MEDIUM (0.70-0.84), LOW (<0.70)
  - calculate_confidence(): Scores an AI response
  - should_require_confirmation(): Decides if user approval needed
  - Auto-apply threshold: 0.90

categorization_service.py (MAIN WORKFLOW)
  Orchestrates entire categorization pipeline:
  1. Rules → Priority-based deterministic matching
  2. Memory → Check stored user decisions
  3. AI → Call AI provider if no match
  4. Fallback → Default category if AI unavailable
  - CategorizationService: Single transaction
  - BatchCategorizationService: Multiple transactions (optimized)

cache.py
  Response caching to reduce AI calls:
  - AICache: Hash-based cache with TTL
  - Cache key: Sanitized merchant + description + rounded amount
  - Saves API calls, latency, and costs

insights_engine.py
  Proactive financial insights:
  - RecurringTransactionDetector: Finds subscriptions
  - InsightsEngine: Generates spending insights
  - 4 insight types: spending_increase, anomaly, subscription, recommendation

repositories.py
  Persistence layer interfaces (to be implemented):
  - RuleRepository, MerchantMemoryRepository, etc.
  - Define what data storage needs to do
  - Concrete implementations use PostgreSQL/Supabase

event_logging.py
  Structured event logging (privacy-safe):
  - Logs: Rule matches, AI requests, cache hits/misses
  - No sensitive data in logs
  - Events: event_type, user_id, timestamp
